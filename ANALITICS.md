# TravLight · Google Analytics 4 계측 문서

랜딩 → 서비스 진입 → 정보 탐색으로 이어지는 사용자 행동을 측정하기 위한 GA4 연동 정리 문서.
코드 구현, 수집 이벤트, GA4 설정, 분석 방법, 그리고 셋업 중 겪은 이슈까지 한곳에 정리한다.

---

## 1. 실험 가설

> 해외여행을 준비하는 2030세대는 비자·세관·날씨·현지 꿀팁 정보를 탐색하며 시간 소모와 신뢰 불안을 겪는다.
> 국가별 정보를 공식 출처와 함께 한 화면에 모아 보여주면,
> ① 랜딩을 본 사용자 중 일정 비율이 **"TravLight 이용하기"** 를 눌러 서비스로 진입하고,
> ② 진입한 사용자는 여행 계획 국가의 정보 **탭을 클릭**해 탐색할 것이다.

이를 **도달 → 전환 → 참여** 3단계 퍼널로 측정한다.

| 단계 | 의미                     | 지표(이벤트)            |
| ---- | ------------------------ | ----------------------- |
| 도달 | 랜딩 페이지 방문         | `page_view` / 방문자 수 |
| 전환 | "이용하기"로 서비스 진입 | `cta_enter`             |
| 참여 | 정보 탭 탐색             | `tab_click`             |

## 2. 사용 도구 및 연결 구성

- **분석 도구**: Google Analytics 4 (gtag.js)
- **측정 ID**: `G-GZ5F1FVYST`
- **배포**: Vercel (`travlight.vercel.app`)
- **측정 ID 주입 방식**: 코드에 하드코딩하지 않고 **Vercel 환경변수**로 주입
  - Key: `VITE_GA_MEASUREMENT_ID`
  - Value: `G-GZ5F1FVYST`
  - Environment: Production
  - Vite는 ㅇ빌드 시점에 환경변수를 코드에 넣으므로, 변수 추가/변경 후 재배포 해야 반영된다.

### 코드 구조

| 파일                                     | 역할                                         |
| ---------------------------------------- | -------------------------------------------- |
| `src/lib/analytics.ts`                   | GA4 초기화 + 커스텀 이벤트 전송 헬퍼         |
| `src/main.tsx`                           | 앱 부트 시 `initAnalytics()` 호출(태그 로드) |
| `src/Root.tsx`                           | (Vercel Analytics 제거됨)                    |
| `src/components/landing/LandingPage.tsx` | CTA 클릭 시 `trackEnter(source)`             |
| `src/App.tsx`                            | 탭 변경 시 `trackTabClick(tab, country)`     |

`analytics.ts`는 측정 ID가 없으면(로컬 개발 등) 아무것도 로드/전송하지 않아 안전하다.
즉 **커스텀 이벤트는 배포본(프로덕션)에서만 집계**된다.

## 3. 수집 이벤트 정의

### 자동 수집 (GA4 기본 / gtag config)

| 이벤트          | 설명              |
| --------------- | ----------------- |
| `page_view`     | 페이지 조회(도달) |
| `first_visit`   | 첫 방문           |
| `session_start` | 세션 시작         |

### 커스텀 이벤트 (직접 계측)

| 이벤트      | 발생 시점                | 매개변수  | 값                                                                   |
| ----------- | ------------------------ | --------- | -------------------------------------------------------------------- |
| `cta_enter` | 랜딩 → 서비스 진입(전환) | `source`  | `hero` · `final_cta` · `nav`                                         |
| `tab_click` | 진입 후 정보 탭 클릭     | `tab`     | `visa` · `prohibited` · `airline` · `packing` · `transport` · `tips` |
|             |                          | `country` | 선택한 나라 이름(예: 일본, 태국 …)                                   |

- `source` — 어느 위치의 CTA에서 전환됐는지
  - `hero`: 랜딩 상단 히어로의 "이용하기"
  - `final_cta`: 랜딩 하단 마지막 "이용하기"
  - `nav`: 상단 바 "바로 시작"
- `tab` — 어떤 정보 카테고리에 관심이 몰리는지
- `country` — 어느 나라 커버리지 수요가 큰지

## 4. GA4 설정 (콘솔 작업)

### 4-1. 맞춤 측정기준 등록 (필수)

커스텀 이벤트의 매개변수(`source`/`tab`/`country`)를 **속성별로 쪼개 보려면** 맞춤 측정기준 등록이 필요하다.

경로: 관리 → **맞춤 정의** → "맞춤 측정기준" 탭 → **맞춤 측정기준 만들기**

| 측정기준 이름 | 범위   | 이벤트 매개변수 |
| ------------- | ------ | --------------- |
| CTA 위치      | 이벤트 | `source`        |
| 탭            | 이벤트 | `tab`           |
| 국가          | 이벤트 | `country`       |

- **이벤트 매개변수 철자**(`source`/`tab`/`country`)는 코드가 보내는 이름과 정확히 일치해야 한다.
- 범위는 3개 모두 **이벤트**.
- 맞춤 측정기준은 등록 이후 쌓이는 데이터부터 쪼개져 보인다(소급 적용 안 됨)

### 4-2. 주요 이벤트(전환) 표시 (선택)

`cta_enter`(원하면 `tab_click`도)를 전환으로 집계하고 싶으면:
관리 → **이벤트** 목록에서 해당 이벤트의 **"주요 이벤트로 표시"** 토글 ON.
(이 버전은 별도 "주요 이벤트" 메뉴 없이 이벤트 목록 안에서 토글로 관리)

## 5. 분석 방법

### 도달

보고서 → **실시간** 또는 **참여도 → 이벤트**의 `page_view` / 방문자 수.

### 전환 + CTA 위치 비교 — 유입경로(Funnel) 탐색

탐색 → **유입경로 탐색**:

1. `page_view` (도달)
2. `cta_enter` (전환)
3. `tab_click` (탐색)

세분화에 **CTA 위치(source)** 를 넣으면 hero / final_cta / nav 중 어디서 더 전환되는지 이탈률로 비교된다.
→ "랜딩 스토리를 다 읽고 눌렀는지, 상단에서 바로 눌렀는지" 파악.

### 참여 — 자유 형식

탐색 → **자유 형식**:

- 이벤트를 `tab_click`으로 필터
- 행(Rows): **탭** 또는 **국가**
- 값(Values): 이벤트 수

→ 비자/세관/항공사/날씨/교통/꿀팁 중 인기 탭, 수요 큰 나라를 표로 확인.

## 6. 지표 → 가설 매핑 요약

| 가설 단계     | 확인 지표               | 보는 곳         |
| ------------- | ----------------------- | --------------- |
| 도달          | 방문자 수, `page_view`  | 실시간 / 이벤트 |

| 국가 수요     | `tab_click` × `country` | 자유 형식       |

## 7. 셋업 중 이슈 & 배운 점

- **측정 ID / 속성 불일치** — 이벤트가 안 잡히던 근본 원인. 코드는 정상이었고, 실시간을 확인하던 속성과 실제 전송 대상 측정 ID가 어긋나 있었다. → 코드가 보내는 `config` 측정 ID와 **실시간을 보는 속성**이 같은지 항상 확인.
- **Vercel Web Analytics의 커스텀 이벤트는 Pro 전용** — 무료(Hobby)에선 방문 수만 보여 퍼널 계측 불가. 그래서 GA4로 전환.
- **VITE\_ 환경변수는 빌드타임 주입** — 변수만 추가하고 재배포 안 하면 반영되지 않는다.
- **광고/추적 차단기·백신·보안 DNS** — 테스트 환경에서 `googletagmanager.com` / `google-analytics.com` 요청이 503 등으로 막히면 이벤트가 전송되지 않는다. 검증은 차단 없는 환경(시크릿 창·다른 기기)에서.
- **맞춤 측정기준은 소급 불가** — 등록 이전 데이터는 매개변수로 쪼개지지 않는다.
- **커스텀 이벤트는 프로덕션에서만** — 로컬(측정 ID 없음)에서는 전송 안 됨.
